# HITL Comment Lock

Never edit/reformat/move/delete text inside `HITL-LOCK` blocks. Append-only outside locks.
Markers: `HITL-LOCK START:<id>` / `HITL-LOCK END:<id>` (use comment syntax for file type)

# Documentation

**Public functions**: `@brief`, `@param`, `@return`, `@note` (thread safety, ISR context).
**File headers**: `@file`, `@brief`, `@author`, `@date`.
**Comments**: explain WHY not WHAT. No commented-out code.

# Workflow

- Plan first → `docs/plans/` → approve → code
- Update plan before next phase. Phases: PENDING → IN PROGRESS → DONE.
- TODOs: `TODO:SCALE`, `TODO:POST-BETA`, `TODO:HARDWARE`, `TODO:SECURITY`, `TODO:ISO`, `TODO:TEST`

# Generated Code

Files in `firmware/ecu/*/cfg/` are generated by `tools/arxml/codegen.py`.
**Never hand-edit generated files.** Modify the generator or DBC source instead.
Generated files have a `/* GENERATED -- DO NOT EDIT */` header.

# Safety

Fail-closed. Generic errors externally. Validate at boundaries. Graceful degradation.
No hardcoded secrets. All faults → safe state transition.
