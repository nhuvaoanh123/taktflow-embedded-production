## Kernel Modules (for CAN on Linux)

```bash
# Required for SIL and POSIX builds
sudo modprobe vcan    # Virtual CAN driver
sudo modprobe can     # CAN core
sudo modprobe can_raw # Raw CAN socket

# For physical USB-CAN adapters
sudo modprobe can_dev
sudo modprobe gs_usb  # Geschwister Schneider USB/CAN (Canable, Cantact)
sudo modprobe peak_usb # PEAK PCAN USB
```

## CI Environment (GitHub Actions)

The CI pipeline runs on `ubuntu-22.04`. All tool versions above are pinned in the workflow files under `.github/workflows/`.

To exactly reproduce CI locally:

```bash
# Match the CI Docker image
docker run -it ubuntu:22.04 bash

# Inside container — install all tools
apt-get update && apt-get install -y \
  gcc make git python3.11 python3-pip \
  lcov cppcheck can-utils libsocketcan-dev \
  libncurses-dev

pip install -r test/sil/requirements.txt
pip install -r tools/arxml/requirements.txt
```

## Generated vs. Hand-Edited Files

**Never hand-edit generated files** — they will be overwritten by the next codegen run.

| Path | Status | Regenerated by |
|------|--------|---------------|
| `generated/` | **Auto-generated** | `python tools/arxml/codegen.py` |
| `firmware/ecu/*/cfg/` | **Auto-generated** | AUTOSAR config tools |
| `firmware/bsw/*/cfg/` | **Auto-generated** | AUTOSAR config tools |
| `firmware/ecu/*/src/` | **Hand-edited** | Developers |
| `firmware/ecu/*/include/` | **Hand-edited** | Developers |
| `firmware/bsw/*/src/` | **Hand-edited** | Developers |
| `test/sil/scenarios/` | **Hand-edited** | Developers |
| `docs/` | **Hand-edited** | Developers |
