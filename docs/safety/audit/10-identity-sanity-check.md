## Identity 7: Safety Analyst (DFA/FMEA)

**Question:** What happens if each module fails?

| Failure Mode | Detection | Safe State | Time to Detect |
|-------------|-----------|------------|----------------|
| Com_MainFunction_Tx stuck | SchM_Timing `g_timing_max_us[0]` > budget | WdgM detects missed checkpoint | 100ms (WdgM cycle) |
| E2E CRC corruption (1 frame) | E2E_Check → E2E_SM stays VALID (windowed) | Frame accepted (tolerated) | N/A |
| E2E CRC corruption (2+ consecutive) | E2E_SM → INVALID → discard + DTC | Signal quality → E2E_FAIL | 20ms (2 × 10ms) |
| CAN bus-off | CanSM L1 recovery (10ms × 5) | Motor stops on RX timeout (100ms) | 50ms (L1) |
| CAN bus-off persistent | CanSM L2 (1s × 10) → permanent BUS_OFF | SC relay kill (200ms) | 10s (L2 max) |
| FZC CAN command timeout | `Com_GetRxPduQuality` → TIMED_OUT | `Swc_FzcSafety` sets FAULT_CAN_BUS_OFF | 100ms |
| DEM event confirmed | FiM inhibits function | SWC skips execution | 100ms (FiM cycle) |
| XCP request to wrong ECU | `Xcp_RxIndication` checks RxPduId, rejects | No impact | Immediate |

**Verdict:** All single-point failures have detection and safe state within FTTI budget.
